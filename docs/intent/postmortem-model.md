# Postmortem Model

Postmortems are aftercare documents for learning and improvement.

## Required Sections

- description
- timeline
- issue resolution
- impact evaluation
- contributing causes analysis
- root cause categories
- lessons learned
- action items
- child incidents
- reviewer state

## Quality Bar

A postmortem is useful when:

- the timeline is chronological and evidence-backed
- impact is described from affected users, teams, service owners, and business capability
- resolution is detailed enough to become a playbook seed
- contributing causes explain trigger, enabling components, and response dynamics
- lessons are reusable by other teams
- action items directly follow from analysis
