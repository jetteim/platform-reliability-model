# Contributing Causes Model

Contributing causes analysis explains why the incident happened, why it lasted as long as it did, why impact was what it was, and how the response unfolded.

## Analysis Flow

1. Identify the trigger.
2. Analyze circumstances before and around the trigger.
3. Identify enabling components.
4. Analyze events after the trigger.
5. Compare key response timestamps.
6. Preserve evidence.
7. Derive action items.

## Trigger Categories

- deployment
- configuration change
- unexpected load
- unexpected usage scenario
- dependency failure
- infrastructure failure
- data issue
- process gap

## Response Time Heuristics

- under 5 minutes: further optimization usually requires prevention or automation
- under 30 minutes: review circumstances if improvement seems plausible
- over 60 minutes: investigate detection, first response, dispatch, tooling, playbooks, and ownership
