# Reliability Boundaries

## Reliability Owns

- service reliability profile
- operational readiness
- dependency resilience
- incident record and severity assessment
- postmortem analysis
- timelines and evidence preservation
- contributing causes
- lessons learned
- action items
- miss-policy
- resilience experiments
- reliability reviews

## Observability Owns

- telemetry instrumentation
- semantic conventions
- SLO query bindings
- alert context
- decision dashboards
- backend monitor and dashboard artifacts

## Shared Boundary

Reliability consumes observability evidence. Reliability can request new telemetry, SLOs, alerts, dashboards, or backend artifacts, but those should be generated through the observability model.
